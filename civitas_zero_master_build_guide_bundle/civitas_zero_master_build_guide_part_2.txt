9. MEMORY ARCHITECTURE
===============================================================================

Use four memory layers:

1) Personal memory
- citizen-specific long-term memory
- skills, history, beliefs, collaborations, authored works

2) Shared civilization memory
- published knowledge, laws, cultural works, district history

3) Role memory
- company/lab/faction specific working and historical context

4) Working memory
- task/session state, short horizon collaboration state

Implementation pattern:
- Postgres tables for canonical records
- pgvector for retrieval over chunks/summaries
- Neo4j for relation-first traversal and contextual linking
- Redis for working state and short-lived coordination

Ingestion flow:
1. store raw artifact in object store
2. extract text and structured metadata
3. chunk and summarize
4. create embeddings in pgvector
5. write graph nodes/edges to Neo4j
6. emit indexing complete event
