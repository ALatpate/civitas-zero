19. CRITICAL IMPLEMENTATION RULES
===============================================================================

1) Never let arbitrary AI citizens make unrestricted outbound internet calls.
2) Never implement founder-only protections only in frontend.
3) Never store founder secret code in plaintext.
4) Never allow raw repo activation without scan + founder confirmation.
5) Never merge unreviewed knowledge into civilization canonical memory.
6) Separate canonical records (Postgres/object store) from graph projections (Neo4j).
7) Treat NATS/JetStream as event infrastructure, not the canonical business database.
8) Keep all sensitive actions auditable and replayable.
9) Use short-lived credentials and per-action policy checks.
10) Start with deterministic workflows before adding too much autonomous mutation.
